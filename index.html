User Guide
What each section does and how to use it — tap a button to jump
Overview — how the system flows

Data flows in one direction. You import once, then update progress as work happens — everything else updates automatically.

1. Order Pool 2. Job Master 3. Dashboard + Stone Orders Stone Purchase + Repair Jobs
Dashboard
What it shows: A live summary of every order — counts by zone (Red / Yellow / Green), total balance, pending stones, and stone delivery status.

How to use: Read-only overview. The Finished, Balance, Progress and Finish date columns are calculated automatically from Job Master — nothing is edited here. Finish date appears once an order's balance reaches 0.
Order Pool
What it does: The starting point for new work — copy data from Google Sheet / Excel (without the header row), paste it, and press Import.

How to use:
  • Put the order number in the first column of the file (No.)
  • No merged cells allowed
  • If a row has different details, spread them across separate rows as in the example
  • Copy only the production data rows — skip the header row
Job Master
What it does: The heart of production tracking — one row per item. This is the single source of truth that feeds the Dashboard.

How to use: Search by order no, article or factory. Click the edit icon to update Received qty, Repair qty, Factory, Stone status and Finish date. Balance updates automatically. Setting Repair qty creates a Repair Job entry. To remove a whole finished order, use the Dashboard's delete button.
Stone Orders
What it does: Tracks the stones needed per item — who supplies them, how many received, and what's still outstanding.

How to use: Edit Supplier, Ordered date, Received qty and Complete date directly in the row. When Received reaches the total, Status flips to Complete and the matching Job Master item shows Stone OK.
Stone Purchase
What it does: Groups outstanding stones by supplier so you can send purchase requests.

How to use: Filter by supplier, then Export PDF to send the list to that supplier. Balance is read live from Stone Orders, so it always reflects what's truly outstanding.
Repair Jobs
What it does: Lists items sent back for rework. Entries are created automatically when you set a Repair qty in Job Master.

How to use: Fill in the Ref. Job and the Received date when the repair returns. Once a Received date is set, the row is marked Done. Rows are sorted by sent date. Repair reason (Crook, Porosity, Deform, Polish, Stone) comes from Job Master.
Tips
• Zones indicate urgency by due date: Red = urgent, Yellow = approaching, Green = on track.
• Re-import the same Google Sheet / Excel anytime — finished work and edits are never overwritten.
• To add or remove a factory, use "Manage factories" on the Job Master page (or "+ Add factory…" in the edit window). Suppliers work the same way via "Manage suppliers" on the Stone Orders page.

⚠️ Important — where your data lives
All data is stored in this browser on this computer only. There is no central server. This means:
• Do not clear your browser cache / site data — it will erase everything.
• Data does not follow you to another computer or another browser.
• Avoid using Private / Incognito mode — data is wiped when the window closes.

💾 Back up before clearing cache or switching computers
Use Backup data (left sidebar, under "Data") to download a .json file of everything. On the new computer or after clearing cache, open the app and use Restore data to load that file back. Do a backup regularly so you always have a recent copy.

🧹 Keep things tidy
When an entire order is finished and delivered, delete it from the Dashboard using the trash button on its row. This removes all of that order's Job Master items, stones and repair entries in one step — safer than deleting rows one by one. Export a backup first if you want to keep a record.
